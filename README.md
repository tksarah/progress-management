
# 進捗管理 PoC

ローカル PC 上で動かすことを前提にした、Tauri ベースのデスクトップアプリ（PoC）です。

**主要技術:** React + Vite (フロントエンド), Tauri (デスクトップランタイム), Rust (ネイティブ部分)

## 概要

このリポジトリはデスクトップアプリとしてローカルの Excel (`.xlsx`) をデータ保存先に使う PoC です。UI から保存先 Excel を選択・作成し、進捗の一覧・登録・更新・報告ドラフト生成などを行います。

## 機能

- Excel ファイルの保存先をアプリ画面から設定
- ファイル参照ダイアログで既存の `.xlsx` を選択
- 新規 Excel ファイルの作成と保存先指定
- 進捗一覧の表示・絞り込み
- 進捗の新規登録・更新
- 報告メモの登録・確認
- 定例報告ドラフトの生成（直近1週間/2週間）
- 会議向け報告の履歴保存と差分抽出
- ステータス別の件数サマリ表示

## 前提（開発者向け）

- Node.js (推奨: 18 以上)
- Rust toolchain（`rustup` で stable をインストール）
- Windows 上での動作を主に想定してテスト
- PowerShell の実行ポリシーで npm が止まる場合、`npm.cmd` を使ってください

## セットアップ（リポジトリルートから）

1. リポジトリをクローンしてルートに移動します。

2. ルートとサブプロジェクトの依存をインストールします（必要に応じて）:

```powershell
# リポジトリルートで
npm install

# サブプロジェクトに個別で入れる場合
cd frontend
npm install
cd ../backend
npm install
cd ..
```

（Windows の PowerShell で問題が起きる場合は `npm.cmd install` のように `npm.cmd` を直接使ってください）

## 開発起動

リポジトリルートで開発モードを起動します:

```powershell
npm run dev
```

上記はプロジェクトに定義された dev スクリプト（Tauri/Vite の同時起動）を実行します。PowerShell で問題がある場合は `npm.cmd run dev` をお試しください。

## ビルド（配布用インストーラ生成）

```powershell
npm run build
```

ビルド後のインストーラは `src-tauri/target/release/bundle/nsis/` 以下に生成されます。

## サンプルデータ投入

確認用のサンプルデータを投入するスクリプトを用意しています（リポジトリルートから実行）:

```powershell
node ./scripts/seed-sample.mjs
```

既定では `progress.xlsx` をプロジェクトの既定保存先（スクリプトの設定に依存）に作成/更新します。

## 構成

- `frontend/`: React ベースの UI
- `src-tauri/`: Tauri 側（Rust）とネイティブ処理（Excel 読み書き）
- `backend/`: 旧 Web PoC の名残（現行のデスクトップ実行では通常不要）
- `scripts/seed-sample.mjs`: サンプルデータ投入スクリプト

## Excel ファイルについて

- 初回起動時はウィザードで新規作成または既存ファイル選択を行います
- 前回指定した Excel が見つかれば自動で読み込まれます。見つからない場合はウィザードが再表示されます
- 共有フォルダの UNC パスも指定可能です
- 既存ファイルを利用する場合、アプリが期待するヘッダー列を持つ必要があります（下記参照）

### 想定ヘッダー例

- RowID
- KPI番号
- カテゴリー
- 担当者名
- 登録日
- 更新日
- ステータス
- ランク
- ディールサイズ
- 社外関係者
- 社内関連部署
- 顧客名
- 内容
- NextAction
- 報告メモ
- 更新者
- Version

## 運用上の注意

- 同一ファイルを複数人で同時編集すると保存競合が発生します。アプリ側では `Version` 列による簡易的な衝突検知を行いますが、運用での同時編集回避を推奨します
- 共有フォルダの権限・ネットワークの安定性によって保存に失敗する場合があります
- 本番導入前に、想定運用環境での同時編集テストと権限確認を行ってください
